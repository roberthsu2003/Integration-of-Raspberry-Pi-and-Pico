# 教師指引

本文件提供教師教學時的建議和注意事項。

## 課程準備

### 硬體準備

**每位學生需要：**
- Raspberry Pi Pico W × 1-2
- Raspberry Pi (共用或個人)
- USB 連接線
- 跳線（選用，用於 UART）
- 麵包板（選用）

**教室需要：**
- WiFi 網路（2.4GHz）
- 投影設備
- 備用 Pico 和線材

### 軟體準備

**課前安裝：**
- Raspberry Pi OS
- Docker 和 Docker Compose
- Python 3.9+
- Thonny IDE
- MicroPython 韌體

**驗證環境：**
```bash
python tools/verify_setup.py
```

## 每日教學重點

### Day 1：Pico 基礎入門

**上午重點：**
- 環境設定（預留充足時間）
- LED 控制基礎
- MicroPython 語法

**常見問題：**
- Thonny 找不到 Pico → 重新安裝韌體
- USB 線不支援資料 → 更換線材
- LED 不亮 → 檢查 Pico W vs Pico

**教學建議：**
- 先示範再讓學生操作
- 準備備用 Pico
- 鼓勵學生互相協助

### Day 2：Pico 進階操作

**上午重點：**
- 感測器讀取
- 資料處理
- 物件導向概念

**常見問題：**
- 溫度讀數異常 → 說明這是晶片溫度
- 資料跳動 → 介紹平滑處理

**教學建議：**
- 讓學生觀察溫度變化
- 討論資料處理的重要性

### Day 3：Pi 基礎與資料庫

**上午重點：**
- Docker 概念
- MongoDB 基礎
- FastAPI 入門

**常見問題：**
- Docker 權限問題 → 加入 docker 群組
- MongoDB 連接失敗 → 檢查容器狀態
- API 無法訪問 → 檢查防火牆

**教學建議：**
- Docker 概念需要清楚說明
- 提供 API 測試工具
- 展示 Mongo Express 介面

### Day 4：MQTT 通訊基礎

**上午重點：**
- MQTT 概念
- Broker 設定
- 主題設計

**常見問題：**
- WiFi 連接失敗 → 檢查 SSID 和密碼
- MQTT 連接逾時 → 檢查 IP 位址
- 訊息沒收到 → 檢查主題匹配

**教學建議：**
- 使用監控工具展示訊息流
- 讓學生設計主題結構
- 強調 QoS 的概念

### Day 5：MQTT 整合與 UART

**上午重點：**
- Pi 訂閱者
- 資料庫整合
- UART 簡介

**常見問題：**
- 資料沒儲存 → 檢查資料庫連接
- 串列埠權限 → 加入 dialout 群組

**教學建議：**
- 展示完整的資料流程
- 比較 MQTT 和 UART
- 討論使用場景

### Day 6：Pi-Pico 整合

**上午重點：**
- 端到端測試
- 系統整合
- 故障排除

**教學建議：**
- 讓學生完整走過流程
- 記錄常見問題
- 分享除錯技巧

### Day 7：多裝置管理

**上午重點：**
- 裝置識別
- 並發處理
- 狀態監控

**教學建議：**
- 展示多個 Pico 同時運作
- 討論擴展性問題

### Day 8：範例專案

**教學建議：**
- 讓學生選擇感興趣的專案
- 鼓勵客製化和創新
- 提供延伸挑戰

### Day 9：綜合專題

**教學建議：**
- 給予充分的實作時間
- 巡迴指導
- 鼓勵展示和分享

## 時間管理

### 彈性時間

每個主題預留 10-15% 彈性時間：
- 環境設定通常需要額外時間
- 學生程度差異
- 突發問題處理

### 加速內容

如果進度超前：
- 提供延伸挑戰題
- 介紹進階概念
- 讓學生協助其他同學

### 延遲處理

如果進度落後：
- 標記選修內容可以跳過
- 簡化練習題
- 課後提供補充材料

## 評量建議

### 每日檢核

使用檢核清單確認學生掌握度：
- 能夠獨立完成基本操作
- 理解核心概念
- 完成練習題

### 專題評分

**評分標準：**
- 功能完整性（40%）
- 程式碼品質（30%）
- 創新性（20%）
- 文件完整性（10%）

## 常見問題處理

### 技術問題

**WiFi 連接：**
1. 確認 SSID 和密碼正確
2. 檢查是 2.4GHz
3. 測試訊號強度
4. 嘗試其他網路

**資料庫連接：**
1. 檢查 Docker 容器狀態
2. 驗證連接字串
3. 檢查防火牆設定
4. 查看容器日誌

**MQTT 問題：**
1. 確認 Broker 運行
2. 檢查 IP 位址
3. 驗證主題匹配
4. 使用監控工具除錯

### 學習問題

**學生跟不上：**
- 提供一對一協助
- 安排同儕協助
- 提供額外練習時間
- 簡化作業要求

**學生太快完成：**
- 提供進階挑戰
- 請協助其他同學
- 鼓勵探索延伸主題
