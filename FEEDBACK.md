# 課程回饋機制

感謝你使用本課程！你的回饋對我們非常重要。

## 📝 回饋方式

### 方式 1：線上問卷（推薦）

我們準備了簡短的問卷，只需 5-10 分鐘：

**學生問卷：** [連結待補充]
**教師問卷：** [連結待補充]

### 方式 2：GitHub Issues

在 GitHub 上提交 Issue：

1. 前往 [Issues 頁面](https://github.com/your-username/pi-pico-integration/issues)
2. 點擊 "New Issue"
3. 選擇適當的標籤：
   - `bug` - 錯誤回報
   - `enhancement` - 功能建議
   - `documentation` - 文件改進
   - `question` - 問題詢問
   - `feedback` - 一般回饋

### 方式 3：電子郵件

直接寄信給我們：
- **一般問題：** [email@example.com]
- **技術支援：** [support@example.com]
- **合作洽詢：** [contact@example.com]

### 方式 4：課堂回饋

如果你是在課堂上使用本教材：
- 向講師提供口頭回饋
- 填寫課堂回饋表
- 參與課後討論

## 🐛 錯誤回報

如果你發現錯誤，請提供以下資訊：

### 程式碼錯誤
- **檔案位置：** 例如 `01_pico_basics/01_led_blink/blink.py`
- **錯誤訊息：** 完整的錯誤訊息
- **重現步驟：** 如何重現這個錯誤
- **預期行為：** 應該要如何運作
- **環境資訊：** OS、Python 版本等

### 文件錯誤
- **檔案位置：** 例如 `README.md` 第 50 行
- **錯誤類型：** 錯別字、連結失效、說明不清等
- **建議修正：** 你認為應該如何修正

### 範例錯誤
- **範例名稱：** 例如「環境監測系統」
- **問題描述：** 哪裡有問題
- **測試環境：** 你的測試環境

1. **收到回饋** → 2-3 個工作天內回覆
2. **評估建議** → 1 週內評估可行性
3. **實作改進** → 納入下一版本規劃
4. **發布更新** → 在 CHANGELOG 中說明
