# 綜合專題模組

Day 9 的學生專題實作指引。

## 專題要求

### 基本要求
- 使用 Pico + Pi 架構
- 包含感測器資料收集
- 使用 MQTT 或 UART 通訊
- 資料儲存到資料庫
- 提供 API 查詢

### 進階要求（選用）
- 多裝置整合
- 自動化控制
- 資料視覺化
- 警報系統
- 創新功能

## 專題流程

### 上午（3 小時）

**09:00-09:30 專題說明**
- 要求和評分標準
- 範例展示
- 分組或個人選擇

**09:30-10:30 專題設計**
- 需求分析
- 架構設計
- 技術選擇
- 實作計畫

**10:30-12:00 開始實作**
- 環境設定
- 基礎功能實作

### 下午（3 小時）

**13:00-14:30 繼續實作**
- 完成核心功能
- 測試和除錯

**14:30-15:30 成果展示**
- 各組/個人展示（5-10 分鐘）
- 功能演示
- 問題討論

**15:30-16:00 課程總結**
- 學習回顧
- 經驗分享
- 後續建議

## 專題範例

### 範例 1：智慧教室監控
**功能：**
- 監控教室溫度和人數（模擬）
- 自動調整空調（LED 模擬）
- 資料記錄和分析

### 範例 2：植物照護系統
**功能：**
- 監控溫度和濕度（模擬）
- 自動澆水提醒（LED 模擬）
- 生長記錄

### 範例 3：能源監控系統
**功能：**
- 監控用電量（模擬）
- 異常用電警報
- 用電統計分析

## 常見問題

**Q: 時間不夠怎麼辦？**
A: 專注於核心功能，進階功能可以標記為未來工作

**Q: 遇到技術問題？**
A: 尋求講師或同學協助，參考現有範例程式碼

**Q: 如何展示創新性？**
A: 加入獨特的功能、改進現有方案、或創意的應用場景
