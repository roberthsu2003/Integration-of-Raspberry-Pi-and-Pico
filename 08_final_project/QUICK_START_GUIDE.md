## 🚀 5 分鐘快速開始

### 步驟 1：選擇起點（2 分鐘）

#### 選項 A：使用模板（推薦新手）
```bash
# 複製專題模板
cp -r 08_final_project/project_template my_project
cd my_project
```

#### 選項 B：參考範例（推薦有想法的學生）
```bash
# 查看範例專題
ls 08_final_project/student_examples/
# 選擇一個範例作為參考
```

#### 選項 C：從零開始（推薦進階學生）
```bash
# 建立自己的專題結構
mkdir my_project
cd my_project
mkdir pico pi docs
```

### 步驟 2：修改配置（2 分鐘）

#### Pico 端配置
編輯 `pico/config.py`：
```python
DEVICE_ID = "my_device_01"        # 改成你的裝置 ID
WIFI_SSID = "your_wifi"           # 改成你的 WiFi
WIFI_PASSWORD = "your_password"   # 改成你的密碼
MQTT_BROKER = "192.168.1.100"     # 改成 Pi 的 IP
```

#### Pi 端配置
編輯 `pi/.env`：
```bash
MQTT_BROKER=localhost
DATABASE_NAME=my_project
```

### 步驟 3：測試運行（1 分鐘）

```bash
# 啟動 Pi 端
cd pi/
docker-compose up -d  # 啟動 MongoDB
python main.py        # 啟動 API

# 上傳 Pico 程式
# 將 pico/ 目錄的檔案複製到 Pico

# 測試連接
curl http://localhost:8000/health
```

## 🐛 常見問題快速解決

### 問題 1：Pico 無法連接 WiFi
```python
# 檢查配置
print(f"SSID: {WIFI_SSID}")
print(f"Password: {WIFI_PASSWORD}")

# 測試連接
import network
wlan = network.WLAN(network.STA_IF)
wlan.active(True)
wlan.connect(WIFI_SSID, WIFI_PASSWORD)
```

### 問題 2：MQTT 連接失敗
```bash
# 檢查 Broker 是否運行
sudo systemctl status mosquitto

# 測試連接
mosquitto_sub -h localhost -t test/#
```

### 問題 3：資料庫連接失敗
```bash
# 檢查 MongoDB 是否運行
docker ps | grep mongo

# 重新啟動
docker-compose restart
```

### 問題 4：API 無法訪問
```bash
# 檢查服務是否運行
ps aux | grep python

# 檢查埠號是否被佔用
lsof -i :8000
```
