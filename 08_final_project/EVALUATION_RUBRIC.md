# 專題評量標準

## 評分總覽

| 評分項目 | 配分 | 說明 |
|---------|------|------|
| 功能完整性 | 40% | 核心功能實現程度 |
| 程式碼品質 | 30% | 程式碼結構和可讀性 |
| 創新性 | 20% | 創意和實用性 |
| 文件完整性 | 10% | 文件和說明品質 |
| **總分** | **100%** | |

## 評分建議

### 給評審的建議

1. **客觀公正**
   - 依據評分標準評分
   - 避免主觀偏見
   - 給予建設性意見

2. **鼓勵創新**
   - 重視創意和想法
   - 不要過度強調技術細節
   - 鼓勵嘗試新方法

3. **考慮學習曲線**
   - 這是學生的第一個專題
   - 重視學習過程
   - 給予改進建議

4. **平衡評分**
   - 不要過於嚴格或寬鬆
   - 考慮整體表現
   - 給予適當鼓勵

### 給學生的建議

1. **專注核心功能**
   - 先完成基本功能
   - 確保功能穩定
   - 再考慮額外功能

2. **重視程式碼品質**
   - 保持程式碼整潔
   - 加入適當註解
   - 遵循命名規範

3. **完善文件**
   - README 要清晰完整
   - 說明安裝和使用步驟
   - 加入展示截圖

4. **展示技巧**
   - 準備展示腳本
   - 測試所有功能
   - 準備備案
   - 保持自信

## 總結

這份評量標準旨在：
- 公平評估學生專題
- 鼓勵創新和實踐
- 提供明確的改進方向
- 培養良好的開發習慣

記住：專題的目的是學習和成長，而不只是分數！
